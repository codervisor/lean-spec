# Internationalization (i18n) Guide

**Every user-facing string MUST have translations in both English and Chinese.**

## Translation File Locations

| Package | English | Chinese |
|---------|---------|---------|
| **UI** | `packages/ui/src/locales/en/*.json` | `packages/ui/src/locales/zh-CN/*.json` |
| **Docs** | `docs-site/docs/` | `docs-site/i18n/zh-Hans/docusaurus-plugin-content-docs/` |

## Files to Update

**UI package** (for web interface strings):
```
packages/ui/src/locales/
├── en/
│   ├── common.json    ← General UI strings
│   ├── errors.json    ← Error messages
│   └── help.json      ← Help text
└── zh-CN/
    ├── common.json    ← 对应英文翻译
    ├── errors.json    ← 错误消息
    └── help.json      ← 帮助文本
```

**Docs site** (for documentation):
```
docs-site/
├── docs/                                    ← English docs
└── i18n/zh-Hans/
    ├── docusaurus-plugin-content-docs/      ← Chinese docs
    ├── TERMINOLOGY_GLOSSARY.md              ← Term mappings
    └── TRANSLATION_STYLE_GUIDE.md           ← Translation guidelines
```

## Checklist (Verify Before Completing Work)

- [ ] Added new key to `en/*.json`? → Add same key to `zh-CN/*.json`
- [ ] Added new doc page? → Create Chinese version in `i18n/zh-Hans/`
- [ ] Modified existing strings? → Update both language files
- [ ] Unsure of translation? → Add `TODO: translate` comment and notify in PR

## Translation Pattern

```json
// packages/ui/src/locales/en/common.json
{
  "newFeature": {
    "title": "New Feature",
    "description": "This feature does X"
  }
}

// packages/ui/src/locales/zh-CN/common.json
{
  "newFeature": {
    "title": "新功能",
    "description": "此功能可以执行 X"
  }
}
```

## Why This Matters

- ~30% of users are Chinese speakers
- Incomplete translations break the user experience
- Keys must match exactly between language files

**⚠️ PRs will be blocked if translations are missing.**
